## When to use this skill

- When organizing or restructuring project files and directories to follow predictable, logical patterns
- When writing or updating README.md files with setup instructions, architecture overviews, or contribution guidelines
- When creating git commit messages following conventional commit format or team standards
- When working with environment configuration files (.env, config files) and ensuring secrets aren't committed
- When managing project dependencies in package.json, requirements.txt, Gemfile, or similar dependency files
- When implementing or documenting code review processes and pull request standards
- When setting up feature flags for incomplete features to avoid long-lived feature branches
- When maintaining changelogs or release notes to track significant project changes
- When defining testing requirements or CI/CD workflows for the project
